§1 Root Cause Summary
The original generateSubCircles function used equal 8.0-mile spacing in both lat and lng axes (not true hex geometry), computed only stepsNeeded=2 from the territory radius, producing a 5×5=25 candidate grid. After the distance filter (dist ≤ 14.77 mi), only 9 circles survived. Nine 5-mile circles cover ~9 × 78.54 = ~707 sq mi theoretical, but with the asymmetric placement and large gaps, effective coverage of the 9.77-mile territory was ~55–60%. The result: ~429 properties returned instead of the expected 800–1,400.

Three compounding bugs:
  1. Equal axis spacing (8.0 mi both axes) instead of true hex geometry (8.66 mi horizontal, 7.5 mi vertical)
  2. Offset applied to column index j (j % 2 !== 0) instead of row index — geometrically wrong
  3. Distance filter cutting boundary circles that are intentionally outside the territory radius but whose 5-mile fetch radius covers boundary properties
